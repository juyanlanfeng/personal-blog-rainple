# 第一部分：你的博客项目

## 1.1 一句话说明

这是一个**个人博客网站**，基于一个开源免费模板 **Fuwari** 搭建。模板作者已经把"房子"盖好了——首页、文章列表、关于页、搜索、深色模式、手机适配等都是现成的。你要做的主要是"搬进去住"：换上你自己的信息、写你的文章、最后发布到网上。

它属于**静态博客**：页面是提前生成好的固定文件，访问快、便宜、安全，非常适合个人写作。

## 1.2 它能做什么

- 用 Markdown 写文章发博客
- 明亮 / 深色模式一键切换
- 主题色可自定义
- 电脑 / 手机 / 平板自适应
- 站内搜索（Pagefind）
- 文章自动生成目录（TOC）
- 平滑动画、页面切换
- RSS 订阅
- 可放头像、横幅、社交链接

## 1.3 文件夹结构（只看你会碰的）

你最常打交道的就这几样：

| 名称 | 是什么 | 你要做什么 |
|------|--------|-----------|
| `src/config.ts` | **博客总设置** | 改标题、副标题、你的名字、头像、简介、社交链接、主题色。**最常改。** |
| `src/content/posts/` | **文章文件夹** | 你写的每篇 `.md` 文章都放这里。 |
| `astro.config.mjs` | 网站网址等配置 | 上线前把里面的网址改成你自己的。 |
| `public/` | 静态资源 | 放图标 favicon、图片等。 |
| `package.json` | 项目信息 + 可用命令清单 | 一般不用动，但值得看一眼"scripts"里有哪些命令。 |

暂时不用动的（知道有就行）：`node_modules/`（自动下载的依赖，很大别删）、`src/components|layouts|pages/`（页面源码，进阶再看）、各种 `.json` / `.config` / `.git`（工具配置）。

## 1.4 常用命令（已改为 npm）

> ⚠️ 重要：你的电脑上 pnpm 下载有问题，**这个项目以后统一用 npm**。下面命令都在项目文件夹里、用 PowerShell 运行。

| 命令 | 作用 |
|------|------|
| `npm install` | 安装项目依赖（第一次或依赖有变动时用） |
| `npm run dev` | 启动本地预览，地址 `http://localhost:4321/`（最常用） |
| `npm run build` | 生成正式上线用的网站文件（输出到 `dist/`） |
| `npm run preview` | 在本地预览"正式版"效果 |
| `npm run new-post -- 文件名` | 新建一篇文章（注意中间那个 `--`，npm 用它来传参数） |
| `npm run check` | 检查代码有没有错误 |
| `npm run format` | 自动整理代码格式 |

**每天启动博客就两步**：

```powershell
cd D:\personal-blog-rainple
npm run dev
```

想关掉预览：在那个窗口按 `Ctrl + C`。

## 1.5 一篇文章长什么样

每篇文章开头有一段"信息卡片"（叫 frontmatter），下面才是正文：

```markdown
---
title: 我的第一篇博客
published: 2026-06-25
description: 这是我用 Astro 博客写的第一篇文章。
tags: [生活, 随笔]
category: 日常
draft: false
---

正文从这里开始，直接用 Markdown 写。

## 这是一个小标题

这是一段普通文字，**这几个字会加粗**。
```

保存后，本地预览会自动刷新，立刻看到效果。

# 第二部分：系统入门网站开发

这一部分不依赖任何前置知识，从"网站到底是什么"讲起，一步步建立完整的认知地图，最后给你一条学习路线。

## 2.1 网站到底是怎么工作的

当你在浏览器输入一个网址、按回车，背后发生了这样一件事：

1. 你的浏览器（**客户端**）向某台一直开着的电脑（**服务器**）发出一个"请求（Request）"，说："把这个页面给我。"
2. 服务器收到后，把对应的网页文件作为"响应（Response）"发回来。
3. 浏览器拿到文件，"渲染"成你看到的页面。

这就是 Web 的核心模型：**请求 → 响应**。理解了这一点，后面很多概念都能挂上钩。

那"网页文件"具体是什么？主要是三种语言写成的文本文件，它们各管一摊：

**HTML —— 结构（骨架）。** 决定页面上有什么内容：这是标题、那是段落、这里是一张图、那里是一个按钮。可以把它想成一个人的骨骼。

**CSS —— 样式（外观）。** 决定这些内容长什么样：颜色、字体、大小、间距、布局、深色模式。相当于皮肤、衣服和妆容。

**JavaScript —— 行为（动作）。** 决定页面能"动"起来：点击按钮有反应、内容能切换、能做搜索、有动画。相当于肌肉和神经，让人能动作和反应。

一个网站，本质上就是浏览器把 HTML、CSS、JavaScript 三者组合起来呈现给你。你这个博客的深色模式切换、搜索、平滑动画，全都是 JavaScript 在背后干活。

## 2.2 静态网站 vs 动态网站

这是一个关键分类，直接决定了一个网站怎么做、怎么部署。

**静态网站**：页面是**提前做好的固定文件**。谁来访问，看到的都是同一份已经准备好的 HTML。优点是快、便宜（甚至免费）、安全、好维护。个人博客、文档站、企业介绍页大多是静态的。**你的 Fuwari 博客就是静态网站**——你写完文章后用 `npm run build` 把所有页面一次性生成好，访客访问的就是这些成品文件。

**动态网站**：页面是**访问时临时生成**的，会根据用户、数据库内容实时变化。比如淘宝（每个人看到的商品/订单不同）、微博（实时刷新的信息流）。它需要服务器在后台运行程序、连接数据库，复杂度和成本都更高。

入门阶段，从静态网站开始是最舒服的路径——而你已经身处其中了。

## 2.3 前端、后端、全栈

你会经常听到这三个词：

**前端（Frontend）**：用户能直接看到、能交互的那部分——也就是浏览器里跑的 HTML / CSS / JavaScript。做博客、做页面外观，都属于前端。

**后端（Backend）**：用户看不到的服务器端逻辑——处理数据、连数据库、做用户登录与权限等。常用语言有 Node.js、Python、Java、Go 等。

**全栈（Full-stack）**：前端后端都做。

你现在做博客，**几乎只涉及前端**（而且大部分还被模板包好了），这正是最适合入门的切入点。

## 2.4 这些工具是干嘛的（把你装过的东西串起来）

你前面安装、运行过一堆工具，这里统一解释它们在整个体系里的位置：

**Node.js**：让 JavaScript 能在浏览器之外（比如你的电脑命令行里）运行的环境。所有现代前端开发工具几乎都跑在 Node.js 上。你装它，就是为了能运行下面这些工具。

**npm（和 pnpm、yarn）**：包管理器，负责帮你下载、管理项目需要的"现成代码包（依赖）"。它们就像"开发界的应用商店"。npm 是 Node.js 自带的；pnpm、yarn 是同类替代品（你这次因为 pnpm 网络问题，改用了 npm，三者作用相同）。

**依赖（dependencies）**：项目运行需要的别人写好的代码包，装在 `node_modules` 文件夹里。你这个博客有 1000 多个依赖——听起来吓人，但都是层层嵌套的小工具，自动管理，不用你操心。

**Astro**：一个"网站框架 / 构建工具"。它把你写的 Markdown 文章、页面组件，自动编译生成成最终的 HTML/CSS/JS 静态文件。Fuwari 就是基于 Astro 做的。类似的框架还有 Next.js、Nuxt、Vite、Hugo 等。

**Git / GitHub**：版本管理工具。Git 记录你每一次代码改动（像游戏存档，可以回退到任意一次），GitHub 是把代码存到云端的网站，也方便配合免费部署。强烈建议尽早学一点点 Git。

**部署（Deploy）**：把做好的网站放上互联网、让别人能访问的过程。常用的免费平台有 Vercel、Netlify、GitHub Pages。

## 2.5 一个完整的开发流程长什么样

把上面的概念连起来，你做博客（以及大多数前端项目）的循环是这样的：

1. **写 / 改**：在编辑器里改 `src/config.ts` 或在 `src/content/posts/` 写文章。
2. **本地预览**：`npm run dev`，浏览器看效果，改完自动刷新。这一步只有你自己能看到。
3. **构建**：满意后 `npm run build`，生成正式的静态文件。
4. **部署**：把构建结果推送到 Vercel / Netlify / GitHub Pages，得到一个公开网址。
5. **（可选）版本管理**：用 Git 把每次改动记录下来、推到 GitHub，既是备份，也方便很多平台自动部署。

记住这个"写 → 预览 → 构建 → 部署"的循环，你就掌握了前端开发的主干节奏。

## 2.7 一条循序渐进的学习路线

下面是给你的建议路线，由近及远、由易到难。不必全部学完才动手——边做博客边学，效果最好。

**阶段 0：先把博客玩熟（你现在在这里）**
目标是建立信心和正反馈。反复做：改 `config.ts` 里的标题/名字 → `npm run dev` 看变化 → 写一篇文章。先不求懂原理，先感受"改了就有反应"。

**阶段 1：Markdown（半天）**
博客写作的基本功，极简单。学会标题、加粗、链接、图片、列表、代码块即可。练习方式就是多写几篇文章。

**阶段 2：HTML + CSS 基础（2~4 周，最重要）**
这是前端的地基。HTML 学常见标签和页面结构；CSS 学选择器、盒模型、Flexbox / Grid 布局、颜色与字体。学完你就能看懂并修改博客的外观了。推荐边学边用浏览器 F12 实时观察真实网页。

**阶段 3：JavaScript 基础（1~2 个月）**
让页面"活"起来的语言，也是入门里最花时间的一块。先学变量、函数、条件、循环、数组与对象、以及如何操作网页元素（DOM）。不用一上来追求精通，够用即可。

**阶段 4：Git 与部署（1 周，可与前面穿插）**
学会用 Git 记录改动、推到 GitHub，并把博客部署到 Vercel 或 GitHub Pages，让全世界能访问。这一步成就感极强。

**阶段 5：理解你的框架 Astro（按需）**
回过头看 Fuwari 的源码结构，理解组件、布局、内容集合是怎么回事。这时你已经有基础，读源码会顺很多。

**阶段 6：往后怎么走（远期）**
如果想继续深入：可学一个前端框架（React / Vue），再了解后端（Node.js + 数据库）走向全栈。但对"做好一个博客"来说，到阶段 4 就完全够用了。

## 2.9 给初学者的几句心里话

- **动手 >> 看书。** 看十个教程不如自己改一行代码看效果。你已经踩过 pnpm 的坑、自己把博客跑起来了——这种"折腾并解决"的经历，正是最值钱的学习。
- **报错是常态，不是失败。** 今天你遇到的一连串报错，每一个都是真实开发里会碰到的。学会读报错、搜报错，比记语法更重要。
- **不懂的文件先别删。** 尤其 `node_modules` 和各种配置文件，删错了重装/恢复都麻烦。
- **小步快跑。** 一次只改一点点、立刻预览，出问题容易定位。别一次改一大片。
- **随时可以找我。** 无论是改配置、写文章、学某个概念，还是又遇到报错，都可以让我带你一步步做。
